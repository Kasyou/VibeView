# 数字产品开发 — 工具类小产品变现

从 CliProject/DigitalProducts 项目提炼。在闲鱼销售原创自动化脚本和桌面小工具。

---

## 核心思路

**Claude 全权负责产品开发，用户只负责上架、发货、收钱。**

小产品要做到"双击即用，零门槛"——目标用户是不懂技术的普通人。

---

## 一、选品策略

### 什么好卖

- 办公效率工具（文件整理、批量处理、格式转换）
- 自动化脚本（填表、采集、批量操作）
- 有 GUI 的工具（比纯命令行好卖 10 倍）
- 解决具体痛点的工具（不要做"平台"或"系统"）

### 什么不碰

- 版权灰色地带（考证资料、课程资源）
- 学术诚信（代写、代做）
- 金融/医疗/法律建议
- 盗版/破解
- 需要资质的服务

### 定价策略

| 价位 | 产品特征 | 目的 |
|------|----------|------|
| 3.99-9.99 | 单一功能、简单工具 | 走量获好评 |
| 19.9-39.9 | 功能完整、有 GUI | 主力盈利 |
| 99-299 | 一对一需求定制 | 高客单价 |

## 二、产品架构模式

### "计划-执行-撤销"三段式

这是做文件/数据处理工具最可靠的架构模式：

```
Engine 层（纯逻辑，不碰文件）:
├── Classifier   — 识别/分类/分析
├── Plan         — 生成操作计划（纯数据结构）
└── Executor     — 执行计划 + 记录日志用于撤销

GUI 层（只负责展示和交互）:
└── App          — 用户界面，调用 Engine
```

### 为什么用"计划"模式

1. **预览安全** — Plan 只是数据结构，不操作实体
2. **撤销简单** — 反向执行 Plan 即可
3. **调试友好** — 可以在不碰真实文件的情况下测试逻辑
4. **出 bug 不丢数据** — 最坏情况也有日志可查

### 撤销机制设计

- 每次操作记录到 JSON 日志文件
- 撤销时倒序还原
- 日志文件放在操作目录下（隐藏文件）
- 撤销后删除日志文件

## 三、GUI 设计原则

### 面向零基础用户

1. **步骤引导式布局** — "第1步、第2步、第3步"降低认知负担
2. **大按钮** — 适合中老年用户，不用精确点击
3. **中文提示** — 所有文案用大白话，不用术语
4. **预览即所见** — 用户在执行前能看到将要发生什么
5. **状态栏** — 始终显示"正在做什么"或"完成了什么"
6. **窗口 640×520 左右** — 确保 1024×768 屏幕正常显示

### GUI 技术选型

- Python: CustomTkinter（比原生 tkinter 好看，零额外学习成本）
- 打包: PyInstaller `--onefile --windowed`
- 输出: 单文件 .exe，用户不用安装任何东西

## 四、闲鱼上架要点

### 标题公式

```
[自动/一键/批量] + 功能描述 + 适用场景 + 免安装
```

### 商品详情结构

1. 主图：工具界面截图 + 大字卖点
2. 使用前后对比（Before/After）
3. 功能介绍（用截图，少写字）
4. 使用说明（3 步以内）
5. 发货方式（网盘链接，永久有效）

### 发货流程

- 百度网盘存放 .exe 文件
- 买家拍下后发网盘链接
- 网盘链接设为永久有效
- 更新版本时替换网盘文件即可

## 六、技术细节备忘

### PyInstaller 打包

```batch
pyinstaller --onefile --windowed --name "产品名" ^
    --add-data "模块;模块" ^
    --hidden-import customtkinter ^
    --clean ^
    main.py
```

### 文件名冲突处理

当目标位置已有同名文件时，自动加序号：`file.txt` → `file(1).txt` → `file(2).txt`

### 错误处理原则

- 文件被占用 → 跳过并告知用户
- 权限不足 → 明确提示
- 空文件夹 → 友好提示"没有可操作的文件"
- 不要静默失败，但也不要弹太多框
